# 报警日志界面设计文档

## 1. 概述

本文档设计手持屏控制器的报警日志界面，用于实时监控报警变量变化并记录报警日志。界面需要满足以下需求：

1. **报警监控**：监控单个报警变量，不同值代表不同报警类型
2. **日志记录**：当报警变量由0变为非0时，添加新日志；当报警值改变时，也添加新日志
3. **显示效果**：新增日志反色显示（白底黑字），选中后按ENTER键标记为已读，取消反色显示
4. **报警恢复**：当报警变量变为0时，所有相关日志取消反色显示（标记为已读）
5. **自动跳转**：新报警产生时自动跳转到报警日志界面
6. **日志容量**：保存10条历史日志，采用循环队列管理
7. **时间戳**：使用系统时间作为报警时间戳

## 2. 架构设计

### 2.1 基于现有菜单系统
报警日志界面将作为现有菜单系统的一个特殊页面，利用以下现有模块：

- **菜单管理器**：管理菜单项和导航
- **显示管理器**：渲染报警日志显示，支持反色显示
- **输入处理器**：处理日志的已读标记操作
- **RS485集成**：基于现有的"页面订阅+跳变检测"机制监控报警变量

### 2.2 报警变量与日志映射
- **单个报警变量**：一个RS485地址，不同值对应不同报警类型
- **报警值映射**：值1-37对应STR_ALARM_E01到STR_ALARM_E37，值255对应STR_ALARM_E255
- **地址示例**：从menu_example.c看，报警变量地址可能是12907（示例中E01的地址）

## 3. 显示逻辑设计

### 3.1 显示格式
每行显示格式：`HH:MM:SS [报警描述]`

示例：
```
16:30:21 E01:急停被按下
16:31:05 E02:外盖未关上
16:32:10 E03:伺服异常
```

### 4.1 特殊按键行为
- **上下键**：在日志条目间导航（标题行也可选中但不执行操作）
- **ENTER键**：标记选中日志为已读，取消反色显示
- **左右键**：翻页（如果日志超过一页显示能力）
- **ESC键**：返回跳转时的菜单
