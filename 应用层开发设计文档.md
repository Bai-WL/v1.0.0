# 手持屏控制器应用层开发设计文档

## 1. 项目概述

### 1.1 项目背景
本项目是基于GD32F303CC微控制器的手持屏控制器，主要面向工业自动化领域，用于设备监控、数据采集和远程控制。

### 1.2 硬件平台

手持屏控制器分为两款：

**WiFi版本（带锂电池）**：
- **主控芯片**: GD32F303CC (Cortex-M4, 256KB Flash, 48KB RAM)
- **显示屏**: JLX192128G (192×128 SPI接口)
- **通信接口**:
  - RS485 Modbus (RTU/TCP协议)
  - WiFi模块 (ATBM6431芯片) - 使用TCP协议
  - USB HID设备接口
- **输入设备**: 正交编码器旋钮
- **电源管理**: 自带3.7V锂电池供电，支持睡眠/唤醒

**有线版本（外接电源）**：
- **主控芯片**: GD32F303CC (Cortex-M4, 256KB Flash, 48KB RAM)
- **显示屏**: JLX192128G (192×128 SPI接口)
- **通信接口**:
  - RS485 Modbus (RTU/TCP协议)
  - USB HID设备接口
- **输入设备**: 正交编码器旋钮
- **电源管理**: 外接电源供电，带纽扣电池用于RTC时钟保持

## 3. 功能模块详细设计

### 3.1 主应用程序框架

#### 3.1.1 系统初始化流程

基于gd32f303xx_bsp驱动代码，系统初始化流程如下：

**所有版本共通的初始化步骤：**
1. **中断优先级配置**：设置NVIC优先级分组为4位抢占优先级
2. **基础外设初始化**：
   - HID输入设备初始化 (`hid_init()`)
   - 系统滴答定时器初始化 (`bsp_SysTickInit()`)
   - ADC初始化 (`bsp_adc_init()`) - 用于电池电压检测
3. **显示初始化**：
   - JLX192128G液晶屏初始化 (`bsp_JLXLcdInit()`)
   - 清屏操作 (`bsp_JLXLcdClearScreen()`)
4. **电源管理初始化**：调用 `pm_power_on()` 函数

**不同版本的差异化初始化：**

**WiFi版本（带锂电池）**：
- **WiFi模块复位引脚配置**：初始化GPIOB Pin2作为WiFi模块复位引脚
- **USB初始化**：根据芯片型号选择DRD USB或标准USB初始化
- **WiFi通信初始化**：
  - Modbus TCP窗口初始化 (`bsp_modbus_tcp_window_init()`)
  - WiFi Modbus主站初始化 (`bsp_ModbusInit()`)，使用TCP协议，波特率115200
- **电源管理特性**：支持锂电池充放电管理，低电量自动关机

**有线版本（外接电源）**：
- **无WiFi模块**：不包含WiFi相关初始化代码
- **USB初始化**：根据芯片型号选择对应的USB初始化
- **电源管理特性**：外接电源供电，支持睡眠/唤醒，带纽扣电池保持RTC时钟

**电源管理初始化详情** (`pm_power_on()` 函数)：
1. **硬件初始化**：初始化GPIOA Pin1作为电源使能引脚
2. **充电检测**：检测USB VBUS状态，进入充电页面或设备初始化
3. **充电页面逻辑**：显示电池充电状态和电量百分比
4. **设备初始化**：完成显示和通信模块初始化

#### 3.1.2 主循环设计

基于最新的代码实现，应用层时间任务已从systick中断移到主循环中执行，采用时间片任务调度模式：

**主循环时间片调度** (在`main()`函数的`while(1)`循环中)：
- 基于系统滴答计时器进行时间片调度，使用无符号减法自动处理计时器回绕
- 每1ms执行`bsp_RunPer1ms()`（RTC实时时钟更新、系统时间刷新、HID按键扫描）
- 每10ms执行`bsp_RunPer10ms()`（ADC采样处理、电源管理节电功能、休眠/关机按键检测）
- 每100ms执行应用层100ms任务预留（供应用层扩展使用）

**电源管理状态机**（版本差异）：
- **WiFi版本（带锂电池）**：
  - 无外部电源时，15分钟无操作自动关机
  - 无外部电源时，5分钟无操作进入休眠
  - 电池电压低于3.1V时强制关机保护
  - 支持USB充电状态检测和管理
  - 支持充电页面显示和电量百分比监测

- **有线版本（外接电源）**：
  - 外接电源供电，无电池电量检测
  - 简化电源管理，无充电页面显示
  - 支持自动休眠/唤醒功能
  - 无低电量关机保护（始终由外接电源供电）

### 5.2 编码规范
1. **命名约定**
   - 全局变量: `g_`前缀 (如 `g_system_state`)
   - 静态变量: `s_`前缀 (如 `s_menu_context`)
   - 函数参数: `_`前缀 (如 `_param`)
   - 常量: 全大写 (如 `MAX_BUFFER_SIZE`)

2. **文件组织**
   - 每个.c文件对应一个.h文件
   - 头文件保护宏: `__MODULE_NAME_H__`
   - 函数声明按模块分组

3. **错误处理**
   - 统一错误码定义
   - 函数返回状态检查
   - 适当的断言使用

## 7. 风险管理和应对措施

### 7.1 技术风险
- **内存不足**: 优化数据结构，使用内存池
- **实时性不足**: 优化任务调度，优先级调整
- **通信不稳定**: 增加重试机制，超时处理

### 7.2 进度风险
- **模块依赖**: 明确接口定义，并行开发
- **技术难点**: 预留缓冲时间，技术预研
- **需求变更**: 保持设计灵活性，迭代开发
