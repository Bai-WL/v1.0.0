# 手持屏控制器菜单导航设计文档

## 1. 概述

### 1.1 设计目标
设计一个适用于GD32F303CC手持屏控制器的菜单导航系统，实现多级菜单的高效浏览、参数编辑和功能操作。系统基于现有的菜单系统框架和BSP驱动，提供直观的用户交互体验。

### 1.2 硬件平台
- **主控芯片**: GD32F303CC (Cortex-M4, 256KB Flash, 48KB RAM)
- **显示屏**: JLX192128G (192×128像素，SPI接口)
- **输入设备**: 6个功能键（F1-F6），映射为上、下、左、右、Enter、Esc功能
- **通信接口**: RS485 Modbus、WiFi模块、USB HID

### 1.3 软件基础
- **菜单系统框架**: 已实现菜单项定义、状态管理、渲染显示
- **显示驱动**: bsp_gd32f303xx_jlx192128g_enhanced.c
- **输入驱动**: drv_hid.c (功能键扫描)
- **多语言系统**: language_resources.c/h
- **电源管理**: drv_pm.c

#### 2.2.2 按键映射层 (Key Mapping Layer)

- 历史记录栈管理

- 对话框和消息框处理

## 3. 导航状态机设计

### 3.1 状态定义
基于现有菜单系统，定义以下导航状态：

```c
typedef enum {
    NAV_STATE_IDLE,              // 空闲状态（系统启动或菜单未激活）
    NAV_STATE_BROWSING,          // 浏览菜单（正常选择状态）
    NAV_STATE_EDITING,           // 编辑参数（数值/开关/列表编辑）
    NAV_STATE_CONFIRM_DIALOG,    // 确认对话框（等待用户确认）
    NAV_STATE_MESSAGE_BOX,       // 消息框（显示信息，自动消失）
    NAV_STATE_LOADING,           // 加载中（等待操作完成）
    NAV_STATE_ERROR,             // 错误状态（显示错误信息）
    NAV_STATE_CHARGING           // 充电状态（WiFi版本特有）
} NavigationState;
```

### 3.2 状态转移图
```
        ┌─────────────────┐
        │     IDLE        │?─────────┐
        └────────┬────────┘          │
                 │ 启动菜单系统       │ 无操作超时（电源管理）
        ┌────────▼────────┐          │
        │    BROWSING     │──────────┘
        └────────┬────────┘
        ┌────────┼────────┐
        │        │        │
┌───────▼────┐  │  ┌─────▼──────┐
│  EDITING   │  │  │CONFIRM_DIALOG│
└───────┬────┘  │  └─────┬──────┘
        │       │        │
        └───────┼────────┼──────────┐
                │        │          │
        ┌───────▼────────▼───────┐  │
        │    MESSAGE_BOX         │  │
        └────────────────────────┘  │
                │                   │
                │ 超时自动返回       │
                └───────────────────┘
```

### 3.3 状态详细说明

#### 3.3.1 BROWSING状态（浏览菜单）
- **功能**: 菜单项选择、进入子菜单、执行动作
- **按键处理**:
  - 上/下键：移动选择光标
  - 左/右键：快速翻页（当菜单项超过一屏时）
  - Enter键：确认选择
  - Esc键：返回上一级
- **长按功能**:
  - Esc键长按：直接返回根菜单
  - Enter键长按：快捷操作（待定义）

#### 3.3.2 EDITING状态（编辑参数）
- **进入条件**: 在BROWSING状态下选择可编辑的菜单项
- **可编辑类型**:
  - TOGGLE（开关）：切换开/关状态
  - VALUE（数值）：调整数值大小
  - LIST（列表）：切换选项
- **按键处理**:
  - 上/下键：单步调整（开关切换、数值±1、列表上下选项）
  - 左/右键：快速调整（数值±10、列表快速切换）
  - Enter键：确认编辑结果
  - Esc键：取消编辑，恢复原值
- **长按功能**:
  - 上/下键长按：连续调整（数值连续变化）
  - 左/右键长按：极速调整（数值大幅变化）

#### 3.3.3 CONFIRM_DIALOG状态（确认对话框）
- **触发条件**: 执行关键操作前（如关机、恢复出厂设置）
- **显示内容**: 提示信息 + "是"/"否"选项
- **按键处理**:
  - 左/右键：在"是"/"否"之间切换选择
  - Enter键：确认当前选择
  - Esc键：取消操作，返回BROWSING状态

#### 3.3.4 MESSAGE_BOX状态（消息框）
- **触发条件**: 显示操作结果、警告信息等
- **特性**: 自动超时返回（默认3秒）
- **按键处理**: 任何按键立即返回BROWSING状态

## 4. 按键映射和事件处理

### 4.1 硬件按键映射
```
F1键 → FUNKEY_UP     (上)
F2键 → FUNKEY_DOWN   (下)  
F3键 → FUNKEY_LEFT   (左/翻页)
F4键 → FUNKEY_RIGHT  (右/翻页)
F5键 → FUNKEY_ENTER  (确认/进入)
F6键 → FUNKEY_ESC    (取消/返回)
```

### 4.2 事件类型定义
```c
typedef enum {
    KEY_EVENT_NONE = 0,         // 无事件
    KEY_EVENT_PRESS,            // 短按（按下后立即释放）
    KEY_EVENT_LONG_PRESS,       // 长按（按住超过1秒）
    KEY_EVENT_REPEAT            // 重复触发（长按后每100ms触发一次）
} KeyEventType;
```

### 4.4 事件队列设计
- **队列大小**: 16个事件（支持事件缓冲）
- **入队时机**: 按键扫描检测到有效事件时
- **出队时机**: 主循环的100ms任务中处理
- **防止事件丢失**: 队列满时丢弃最旧事件

## 5. 导航逻辑设计

### 5.1 菜单树结构
基于现有MenuItem数据结构，菜单项通过链表形式组织：
```c
typedef struct MenuItem {
    MenuItemTypes type;        // 菜单项类型
    uint16_t id;               // 唯一ID
    uint16_t parent_id;        // 父菜单ID
    uint16_t first_child_id;   // 第一个子项ID
    uint16_t next_sibling_id;  // 下一个兄弟项ID
    // ... 其他字段
} MenuItem;
```

**导航关系**:
- 每个SUBMENU类型的菜单项都有一个`target_menu_id`指向子菜单
- 菜单项通过`parent_id`、`first_child_id`、`next_sibling_id`形成树形结构
- 根菜单的`parent_id`为0，`first_child_id`指向主菜单第一项

### 5.2 历史记录管理
**历史栈设计**:
- **栈大小**: 8级（支持最多8级菜单深度）
- **栈操作**:
  - 进入子菜单时，当前菜单ID入栈
  - 返回上一级时，栈顶菜单ID出栈
  - 直接返回根菜单时，清空历史栈
- **栈溢出处理**: 循环覆盖最旧记录（先进先出）

### 5.3 滚动和分页机制
**屏幕限制**: 192×128像素，最多显示5个菜单项（每项16像素高）

**滚动逻辑**:
1. 当菜单项总数 ≤ 5时：不显示滚动条，全部显示
2. 当菜单项总数 > 5时：
   - 显示当前可见范围（scroll_position到scroll_position+4）
   - 上下键移动时自动调整滚动位置
   - 左右键实现快速翻页（每次翻一页，5个项）

**滚动位置计算**:
```c
// 当向下移动时
if (item_index >= menu_ctx.scroll_position + visible_items) {
    menu_ctx.scroll_position = item_index - visible_items + 1;
}

// 当向上移动时  
if (menu_ctx.scroll_position > 0) {
    menu_ctx.scroll_position--;
}
```

### 5.4 编辑模式处理
**编辑流程**:
1. 在BROWSING状态下选择可编辑项（TOGGLE/VALUE/LIST）
2. 进入EDITING状态，当前项进入编辑模式
3. 根据类型使用不同编辑逻辑：
   - **TOGGLE**: 上/下键切换状态
   - **VALUE**: 上/下键单步调整，左/右键快速调整
   - **LIST**: 上/下键切换选项
4. Enter键确认修改，执行回调函数
5. Esc键取消修改，恢复原值

**编辑边界处理**:
- 数值类型：检查min_value和max_value边界
- 列表类型：循环选择（到达末尾后回到开头）

### 5.5 对话框处理
**确认对话框流程**:
1. 调用`menu_show_confirm()`显示对话框
2. 进入CONFIRM_DIALOG状态
3. 用户选择"是"或"否"
4. 执行对应回调函数
5. 返回原状态

**消息框流程**:
1. 调用`menu_show_message()`显示消息
2. 进入MESSAGE_BOX状态
3. 启动超时定时器（默认3000ms）
4. 超时或用户按键后返回原状态

## 6. 与驱动层集成

### 6.1 显示驱动集成
**显示更新策略**:
- **部分更新**: 只更新变化的区域（菜单项、状态栏）
- **全屏更新**: 状态切换时（如进入编辑模式）
- **双缓冲**: 使用显示缓冲区，避免闪烁

**渲染调用链**:
```
main循环 (100ms)
    ↓
menu_handle_timer() 检查重绘标志
    ↓
render_full() 完整渲染
    ├─ render_header_internal() 标题栏
    ├─ 循环渲染可见菜单项
    └─ render_footer_internal() 状态栏
```

### 6.2 按键驱动集成
**按键扫描流程**:
```
1ms中断任务
    ↓
bsp_RunPer1ms()
    ↓
hid_get_funkey() 获取原始按键值
    ↓
key_scan_state_machine() 状态机处理
    ↓
生成InputEvent事件
    ↓
input_queue_push() 事件入队
```

**主循环事件处理**:
```c
// 100ms任务中
InputEvent event;
while (input_queue_pop(&event)) {
    menu_handle_key_event(event.key, event.type);
}
```

### 6.3 多语言系统集成
**语言切换**:
- 通过`menu_set_language()`设置当前语言
- 渲染时通过`get_string()`获取对应语言字符串
- 语言资源存储在`language_resources.c`中

**文本换行处理**:
- 根据中英文字符宽度不同计算文本宽度
- 自动换行算法确保文本不超过显示区域
- 长文本支持显示省略号

### 6.4 电源管理集成
**自动休眠/关机**:
- WiFi版本：15分钟无操作自动关机，5分钟无操作进入休眠
- 有线版本：支持休眠/唤醒功能
- 编辑模式下暂停休眠计时

**低电量处理**:
- 电池电压低于3.1V时强制关机
- 充电状态下显示充电页面
- 电量显示在标题栏

## 8. 导航流程示例

### 8.1 典型用户操作流程
**场景**: 用户修改系统参数M01（开关类型）

1. **启动**: 系统启动，显示主菜单（BROWSING状态）
2. **导航**: 按下键选择"系统参数"，按Enter进入
3. **选择**: 按下键选择"M01"项，按Enter进入编辑
4. **编辑**: 进入EDITING状态，按下键切换开关状态
5. **确认**: 按Enter确认修改，返回BROWSING状态
6. **返回**: 按Esc返回上一级菜单

**事件序列**:
```
DOWN(PRESS) → 选择"系统参数"
ENTER(PRESS) → 进入子菜单
DOWN(PRESS) → 选择"M01"
ENTER(PRESS) → 进入编辑模式
DOWN(PRESS) → 切换开关状态
ENTER(PRESS) → 确认修改
ESC(PRESS) → 返回上级
```

### 8.2 快速操作流程
**场景**: 用户需要快速返回到主菜单

1. **当前状态**: 在三级子菜单中
2. **快速返回**: 长按Esc键1秒
3. **结果**: 直接返回到根菜单（主菜单）

### 8.3 数值编辑流程
**场景**: 用户修改参数P00（数值类型，范围0-9999）

1. **进入编辑**: 选择P00项，按Enter
2. **单步调整**: 按上/下键，每次±1
3. **快速调整**: 按左/右键，每次±10
4. **连续调整**: 长按上/下键，连续变化
5. **确认/取消**: Enter确认，Esc取消
